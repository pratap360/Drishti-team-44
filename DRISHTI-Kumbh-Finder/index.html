③ Cross-center duplicate detection

The core failure: the same person reported at two centers shows up as two open cases — families wait while a "found" record already exists elsewhere. DRISHTI clusters likely-same records across different centers by attribute similarity so an operator can link & resolve them. (~8% of cases in this dataset are cross-center duplicates.)

ℹ️ How DRISHTI works & design choices

DRISHTI turns ten disconnected lost-and-found centers into one searchable registry. The matching engine scores any two records on gender, age band, language, last-seen proximity (real GPS), name similarity and report time, producing a confidence score used for both search ranking and cross-center duplicate linking.

Built around the judging criteria:

  • Deployability — runs from a single HTML file, fully offline (data embedded, map drawn from coordinates, no tiles/CDN needed for the core flow).
  • Real-world fit — closes the cross-center search gap and the 8% duplicate problem, the dataset's stated core failures.
  • UX for phoneless / non-literate — operator-driven kiosk intake, attribute-based (no app on the missing person), language captured, big touch targets.
  • System design — handles missing name (15%) & mobile (20%), partial queries, and offline operation; a face photo is an optional enhancement, never required.
  • Responsible data — synthetic records only; mobile numbers masked on load; nothing leaves the device.

Face matching uses pre-trained in-browser models (no training-from-scratch) and only activates when a photo is attached. Data: Kumbhathon Innovation Foundation (locations) + synthetic missing-person set (Claude Impact Lab, Mumbai 2026).
